דוח סיכום פרויקט: בניית שרת Express וייעול קוד (CRUD & Best Practices)

--- חלק א': מימוש עקרונות ה-CRUD הבסיסיים ---
בשלב הראשון של הפרויקט, הוקם שרת יישומי בסיסי באמצעות Express המנהל שלוש ישויות מידע בזיכרון (In-Memory): קורסים (Courses), תלמידים (Students) ורישומים לקורסים (Enrollments). 

עבור כל אחת מהישויות מומשו פעולות ה-CRUD המלאות:
1. Create (יצירה): נתיבי POST ליצירת איברים חדשים ומניעת כפילויות (למשל, מניעת רישום כפול של תלמיד לאותו קורס).
2. Read (קריאה): נתיבי GET לקבלת רשימה מלאה או שליפת פריט בודד על פי מזהה ייחודי (ID).
3. Update (עדכון): נתיבי PUT לעדכון דינמי של שדות קיימים תוך שמירה על המידע הקיים בשדות שלא נשלחו לעדכון.
4. Delete (מחיקה): נתיבי DELETE למחיקת פריטים, כולל מנגנון מחיקה משורשרת (Cascading Delete) – מחיקת קורס או תלמיד מוחקת באופן אוטומטי את הרישומים המשויכים אליהם כדי למנוע נתונים יתומים (Orphaned Data).

--- חלק ב': שדרוג וייעול הקוד (Refactoring) ---
בשלב השני, בוצע שדרוג משמעותי לקוד במטרה ליישם שיטות עבודה מומלצות בתעשייה (Best Practices) תוך שמירה על מבנה קובץ ריצה אחוד (index.js):

1. ייעול מציאת מזהים (עקרון DRY - Don't Repeat Yourself):
- הבעיה בקוד המקורי: בכל נתיב יצירה (POST) נכתבה מחדש הלוגיקה למציאת ה-ID הגבוה ביותר במערך לצורך יצירת ה-ID הבא בתור.
- הפתרון: יצירת פונקציית עזר גנרית אחת בראש הקובץ המשרתת את כל הישויות:
  const getNextId = (array) => {
    return array.length > 0 ? Math.max(...array.map(item => item.id)) + 1 : 1;
  };
- התועלת: מניעת כפילויות קוד, הגברת הקריאות וצמצום פוטנציאל השגיאות.

2. העברת ולידציות ל-Middleware מרכזי:
- הבעיה בקוד המקורי: בדיקת קיומם של שדות חובה (כמו שם קורס או מזהה תלמיד) בוצעה באופן ידני בתוך גוף הפונקציה של כל נתיב ונתיב.
- הפתרון: יצירת פונקציית Middleware גנרית בשם validateFields המקבלת מערך של שמות שדות החובה הנדרשים:
  const validateFields = (requiredFields) => {
    return (req, res, next) => {
      const missingFields = [];
      for (const field of requiredFields) {
        if (req.body[field] === undefined || req.body[field] === null || req.body[field] === '') {
          missingFields.push(field);
        }
      }
      if (missingFields.length > 0) {
        const error = new Error(`השדות הבאים חסרים בגוף הבקשה: ${missingFields.join(', ')}`);
        error.status = 400;
        return next(error);
      }
      next();
    };
  };
- התועלת: הפרדה מוחלטת בין שלב בדיקת תקינות הקלט (Validation) לבין ביצוע הפעולה העסקית בנתיב (Separation of Concerns). אם שדה חסר, הבקשה נעצרת מיד ונשלחת לטיפול השגיאות המרכזי.

3. ניהול שגיאות קצה לקצה:
כל הנתיבים עטופים בבלוקים של try/catch המעבירים כל שגיאה פוטנציאלית (כמו פריט שלא נמצא או שגיאת קלט) באמצעות next(err) אל ה-Error Handler המרכזי של Express בקצה הקובץ, המציג שגיאות ידידותיות ומובנות למשתמש.
